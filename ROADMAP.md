### 🗺️ AuraConv-Multi-NewDSP Roadmap

| Version | Type | Feature Title | Technical Description | Acoustic & Workflow Benefits | DSP Implementation Challenge | Complexity |
| :--- | :--- | :--- | :--- | :--- | :--- | :---: |
| **v4.1.0** | Minor | **Selectable Windowing Functions** | Integrates alternative window shapes (Hann, Hamming, Blackman-Harris) to weight the coefficient values of the moving average buffer. | Smooths the raw "boxcar" filter’s frequency response, mitigating the hollow, metallic ringing (sidelobe resonance) typical of basic moving averages. | Requires storing dynamic window weight arrays inside the `AudioWorklet` thread and scaling coefficient values on the fly. | **Medium** |
| **v4.2.0** | Minor | **In-Browser WAV Exporter** | Connects a dynamic buffer-writing recorder directly to the master output node to capture and encode stereo output. | Allows sound designers to export and save their live performances and automation sweeps directly to uncompressed 24-bit WAV files. | Managing synchronous, thread-safe memory writes of the `AudioWorklet` output stream without inducing system latency or dropouts. | **Low** |
| **v4.3.0** | Minor | **Web MIDI API Integration** | Adds MIDI-In mapping capabilities for master faders, individual channel gains, and preset triggers using `navigator.requestMIDIAccess`. | Allows physical controller integration (knobs, faders) for a tactile, hardware-like performance and automation experience. | Ensuring low-latency thread safety when translating high-speed MIDI CC control messages to smoothed parameter changes in the Worklet. | **Medium** |
| **v5.0.0** | Major | **Linkwitz-Riley Multi-Band Crossover** | Integrates a phase-correct Linkwitz-Riley 3-band crossover network, splitting input signals into Low, Mid, and High bands. | Allows users to isolate and blur specific bands (e.g., blurring high-mid textures while keeping low frequencies tight and completely punchy). | Designing phase-correct crossover filters without creating group-delay anomalies when summing parallel chains back to master. | **High** |
| **v5.1.0** | Minor | **WebGL Spectral Waterfall Spectrogram** | Replaces the basic 2D Canvas with a high-performance, 3D WebGL visualizer depicting real-time time-frequency-amplitude sweeps. | Provides high-resolution visual analytics showing the exact transient smearing across the frequency spectrum over time. | Constructing an efficient WebGL pipeline that shares FFT analysis data with the audio thread without starving the graphics pipeline. | **Medium** |
| **v6.0.0** | Major | **WASM-SIMD DSP Acceleration** | Porting the core moving average processing loops from native JavaScript to WebAssembly (WASM) utilizing Rust or C++. | Lowers the CPU footprint of processing 100+ active parallel channels, extending the battery life and stability of older mobile devices. | Managing thread-safe, high-speed memory buffers (`SharedArrayBuffer`) and compiling SIMD vectorization routines for browsers. | **High** |
